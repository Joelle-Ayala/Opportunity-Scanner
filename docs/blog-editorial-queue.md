# Opportunity Scanner Blog Editorial Queue

## Purpose

This internal queue merges the founder's 36-topic list with the 20 live resources in lib/marketingContent.ts. It removes duplicate search intent, preserves every live URL, and adds the local-business credibility topic requested for launch planning.

The result is 43 canonical resources:

- 16 existing articles that already cover the intended topic.
- 4 refreshes where a live article should absorb or sharpen a founder topic rather than create a competing URL.
- 23 net-new articles, including the added local-business credibility article.

## Status and sequencing rules

- **Existing:** live resource; maintain and add a full proof/visual/repurposing package when scheduled.
- **Refresh:** keep the live slug and expand or reposition the article. Do not create a second URL for the founder-title variant.
- **Net-new:** no live canonical resource currently covers the search intent.
- **Launch:** part of the first public content set or a direct support asset for it.
- **Next:** the next source, workflow, or ICP cluster after the launch set.
- **Expansion:** vertical or long-tail growth after the core content system is proven.
- **Maintain:** useful live support content that does not need to block launch.

Founder numbers refer to the numbered list in the founder brief. A dash means the resource came from the existing library rather than the founder list.

## Editorial principles

1. Keep one canonical article per search intent. Use refreshes and internal links instead of near-duplicate posts.
2. Prioritize articles that explain the core promise: public records become buyer, partner, funding, and workflow actions.
3. Publish source explainers only when they teach a business-development workflow, not merely how to navigate a portal.
4. Make the action layer visible in every article: source, target, revenue motion, contact path, and next best action.
5. Use only verified public facts. Store source URL, retrieval date, metric definition, and chart transformation notes in the internal package.
6. Keep production labels, prompts, UTM plans, and repurposing notes out of public article copy.
